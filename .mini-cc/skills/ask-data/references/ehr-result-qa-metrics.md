# 指标口径

## 接口调用指标

来源表：`public.dx_lxj_call_dzbl`。

- 单接口调用次数：`成功数 + 失败数`
- 总成功次数：13 个接口成功数字段之和。
- 总失败次数：13 个接口失败数字段之和。
- 总调用次数：总成功次数 + 总失败次数。
- 接口成功率：`总成功次数 / 总调用次数`

该表是全量表，不按数据期别过滤。

## 接口开通状态指标

来源表：`public.dx_lxj_call_dzbl_comp`。

- 数据期别字段：`"统计日期"`
- 某接口是否完成：对应接口字段值为“是”。
- 已完成接口数量：`"已完成接口数量"`
- 未完成接口数量：`"未完成接口数量"`
- 较前一日新增完成接口数量：`"较前一日新增完成接口数量"`
- 全部完成机构：`"已完成接口数量" = 13`
- 未全部完成机构：`"已完成接口数量" < 13`

默认统计 13 个接口：4301、4302、4401、4402A、4501A、4502A、4503、4504、4505、4506、4601、4602、4701。

## 上传情况指标

来源表：`public.dx_lxj_dzbl_scqk`。

- 数据期别字段：`"数据期别"`
- 机构上传总数量：`"上传总数量"` 安全转 numeric 后统计。
- 区划上传总数量：按 `"区划"` 分组汇总上传总数量。
- 机构等级上传总数量：按 `"机构等级"` 分组汇总上传总数量。
- 各类电子病历上传数量：对对应中文数量字段安全转 numeric 后统计。

数量字段为 varchar，必须使用安全转换：

```sql
CASE
  WHEN NULLIF(TRIM(<数量字段>), '') ~ '^[0-9]+(\.[0-9]+)?$'
  THEN NULLIF(TRIM(<数量字段>), '')::numeric
  ELSE 0
END
```

## 无效上传汇总指标

来源表：`public.dx_lxj_dzbl_scqk_not`。

业务口径：电子病历上传数据中的就诊 ID、人员编号必须与结算记录对应字段一致；匹配不上即无效上传，需要通报。

- 数据期别字段：`"数据期别"`
- 机构无效上传总数量：`"上传总数量"` 安全转 numeric 后统计。
- 区划无效上传总数量：按 `"区划"` 分组汇总无效上传总数量。
- 机构等级无效上传总数量：按 `"机构等级"` 分组汇总无效上传总数量。
- 需要通报机构数：无效上传总数量大于 0 的机构数。

注意：该表字段名仍是“上传数量”，但在本表中表示“无效上传数量”。

## 无效上传明细指标

来源表：`public.dx_lxj_dzbl_total_main_not_mx`。

- 数据期别字段：`date_period`
- 无效上传明细数：`COUNT(*)`
- 涉及机构数：`COUNT(DISTINCT fixmedins_code)`
- 涉及人员数：`COUNT(DISTINCT psn_no)`
- 涉及就诊数：`COUNT(DISTINCT mdtrt_id)`
- 按接口统计：按 `interface` 分组 `COUNT(*)`
- 按机构统计：按 `fixmedins_code`、`fixmedins_name` 分组 `COUNT(*)`

## 比率指标

- 接口成功率：`成功次数 / NULLIF(成功次数 + 失败次数, 0)`
- 接口平均完成率：`SUM("已完成接口数量") / NULLIF(COUNT(*) * 13, 0)`
- 无效上传占比：`无效上传总数量 / NULLIF(上传总数量, 0)`，仅在用户明确要求将 `scqk` 和 `scqk_not` 对比时计算。

计算比率时必须使用 `NULLIF(分母, 0)` 防止除零。

## 排序口径

- 机构明细默认按序号升序：`"序号"`。
- 用户问“最多、最高、排名、TopN、前几名、最大、最少、最低”等排名类问题时，按目标指标值排序。
- 用户问“分别是多少、明细、清单、情况、列表、有哪些”等明细类问题时，默认按业务序号升序排序。
- 上传数量、无效上传数量只有在排名类问题中才默认按数量降序。
- 无效上传明细默认按机构编码、接口、就诊 ID 排序。

## 月份过滤口径

- `dx_lxj_call_dzbl_comp."统计日期"`、`dx_lxj_dzbl_scqk."数据期别"`、`dx_lxj_dzbl_scqk_not."数据期别"`、`dx_lxj_dzbl_total_main_not_mx.date_period` 均按日粒度 `yyyyMMdd` 使用。
- 用户按月份查询时，使用半开区间过滤整月，例如 2026 年 6 月：
  - `"统计日期" >= '20260601' AND "统计日期" < '20260701'`
  - `"数据期别" >= '20260601' AND "数据期别" < '20260701'`
  - `date_period >= '20260601' AND date_period < '20260701'`
- 不要生成 `"数据期别" = '202606'` 或 `date_period = '202606'`。
- 用户按月或时间范围查询机构清单时，必须按机构去重或汇总：
  - 只列机构：`GROUP BY "机构编码"` 或 `GROUP BY fixmedins_code`。
  - 机构名称：使用 `MAX("机构名称")` 或 `MAX(fixmedins_name)`。
  - 机构级结果表排序：使用 `ORDER BY MIN("序号") ASC`。
  - 明细表无 `"序号"` 时，使用机构编码排序，或关联汇总表取 `MIN("序号")` 后排序。

## 需确认项

- `dx_lxj_call_dzbl` 为全量表，若用户要求某数据期别调用情况，需要确认是否存在对应快照表。
- `dx_lxj_dzbl_scqk_not` 表名带 `_not`，但业务口径是“无效上传”，不是“未上传”。
- `dx_lxj_dzbl_total_main_not_mx` 只能说明上传记录与结算记录匹配不上，不能直接推断真实业务未发生。
