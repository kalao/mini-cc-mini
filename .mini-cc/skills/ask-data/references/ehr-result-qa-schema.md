# 表结构

当前电子病历结果表问数默认使用以下表：

- `public.dx_lxj_call_dzbl`：接口调用情况全量表。
- `public.dx_lxj_call_dzbl_comp`：医疗机构接口开通状态每日统计表。
- `public.dx_lxj_dzbl_scqk`：上传情况机构汇总表。
- `public.dx_lxj_dzbl_scqk_not`：无效上传情况机构汇总表。
- `public.dx_lxj_dzbl_total_main_not_mx`：无效上传明细表。

## 接口调用情况全量表

`public.dx_lxj_call_dzbl`：

- 机构编码：`"机构编码"`
- 机构名称：`"机构名称"`
- 机构等级：`"机构等级"`
- 区划：`"区划"`
- 序号：`"序号"`
- 13 个接口成功数、失败数：
  - `"trns_4301成功数"`、`"trns_4301失败数"`
  - `"trns_4302成功数"`、`"trns_4302失败数"`
  - `"trns_4401成功数"`、`"trns_4401失败数"`
  - `"trns_4402a成功数"`、`"trns_4402a失败数"`
  - `"trns_4501a成功数"`、`"trns_4501a失败数"`
  - `"trns_4502a成功数"`、`"trns_4502a失败数"`
  - `"trns_4503成功数"`、`"trns_4503失败数"`
  - `"trns_4504成功数"`、`"trns_4504失败数"`
  - `"trns_4505成功数"`、`"trns_4505失败数"`
  - `"trns_4506成功数"`、`"trns_4506失败数"`
  - `"trns_4601成功数"`、`"trns_4601失败数"`
  - `"trns_4602成功数"`、`"trns_4602失败数"`
  - `"trns_4701成功数"`、`"trns_4701失败数"`

该表是全量表，没有数据期别字段。

注意：接口调用字段带 `trns_` 前缀，且 `4402a`、`4501a`、`4502a` 中的 `a` 为小写。

## 接口开通状态每日统计表

`public.dx_lxj_call_dzbl_comp`：

- 机构编码：`"机构编码"`
- 机构名称：`"机构名称"`
- 机构等级：`"机构等级"`
- 区划：`"区划"`
- 序号：`"序号"`
- 数据期别：`"统计日期"`
- 13 个接口状态：`"trns_4301"`、`"trns_4302"`、`"trns_4401"`、`"trns_4402a"`、`"trns_4501a"`、`"trns_4502a"`、`"trns_4503"`、`"trns_4504"`、`"trns_4505"`、`"trns_4506"`、`"trns_4601"`、`"trns_4602"`、`"trns_4701"`
- 已完成接口数量：`"已完成接口数量"`
- 未完成接口数量：`"未完成接口数量"`
- 较前一日新增完成接口数量：`"较前一日新增完成接口数量"`

接口状态字段取值通常为“是/否”。

注意：接口状态字段带 `trns_` 前缀，且 `4402a`、`4501a`、`4502a` 中的 `a` 为小写。

## 无效上传情况汇总表

`public.dx_lxj_dzbl_scqk_not` 字段结构与 `dx_lxj_dzbl_scqk` 基本相同：

- 序号：`"序号"`
- 区划：`"区划"`
- 机构等级：`"机构等级"`
- 机构编码：`"机构编码"`
- 机构名称：`"机构名称"`
- 无效上传总数量：`"上传总数量"`
- 数据期别：`"数据期别"`
- 各类字段名仍为“上传数量”，但在本表中表示对应类别的无效上传数量。

## 无效上传明细表

`public.dx_lxj_dzbl_total_main_not_mx`：

- 接口/数据类型：`interface`
- 类型：`l_type`
- 机构编码：`fixmedins_code`
- 机构名称：`fixmedins_name`
- 就诊 ID：`mdtrt_id`
- 人员编号：`psn_no`
- 就诊流水号：`mdtrt_sn`
- 数据期别：`date_period`

该表用于定位具体需要通报的无效上传记录。

## 关联建议

- 机构级表之间可按机构编码和数据期别关联：
  - `dx_lxj_call_dzbl_comp."机构编码" = dx_lxj_dzbl_scqk."机构编码"`
  - `dx_lxj_call_dzbl_comp."统计日期" = dx_lxj_dzbl_scqk."数据期别"`
- 无效上传汇总和明细可按机构编码和数据期别关联：
  - `dx_lxj_dzbl_scqk_not."机构编码" = dx_lxj_dzbl_total_main_not_mx.fixmedins_code`
  - `dx_lxj_dzbl_scqk_not."数据期别" = dx_lxj_dzbl_total_main_not_mx.date_period`

默认不要跨表关联，除非用户明确要求对比分析。
